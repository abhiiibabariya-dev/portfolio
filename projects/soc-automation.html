SOC Automation Lab — Wazuh + TheHive + Shuffle

A production-grade SOC automation stack that reduced manual triage time by 80% through playbook-driven response.

Personal Lab / Portfolio
Duration
6 weeks
Role
Architect · Detection · Automation

Overview

Designed and deployed a fully integrated SOC automation environment: Wazuh for log ingestion and detection, TheHive for case management, and Shuffle for SOAR orchestration. The lab ingests real endpoint & network telemetry, enriches indicators via VirusTotal and AbuseIPDB, and auto-creates cases with pre-populated observables and playbook-driven response steps.

Problem

  • Manual alert triage was consuming disproportionate analyst time on repetitive enrichment steps.
  • Alerts, cases, and enrichment lived in disconnected tools with no shared context.
  • No consistent way to measure playbook effectiveness or false-positive rate over time.

Phases

01
Architecture & Ingestion
  • Deployed Wazuh manager + indexer + dashboards on a hardened Ubuntu host.
  • Onboarded Windows, Linux, and Sysmon agents; ingested Suricata NIDS events via rsyslog.
  • Structured indexes for hot/warm tiers and configured retention aligned with case-review needs.
02
Detection Engineering
  • Authored custom Wazuh rules for PowerShell obfuscation, WMI abuse, and scheduled-task persistence.
  • Mapped every rule to MITRE ATT&CK sub-technique for coverage tracking.
  • Tuned noisy defaults; whitelisted expected admin activity to slash false positives.
03
TheHive + Cortex Case Management
  • Configured TheHive with a Wazuh webhook — every high-severity alert auto-generates a case.
  • Connected Cortex analyzers (VirusTotal, AbuseIPDB, MISP) to enrich observables on ingest.
  • Built case templates with pre-populated tasks aligned to the PICERL lifecycle.
04
Shuffle SOAR Playbooks
  • Built playbooks: phishing triage, brute-force response, malware execution, and IOC sweep.
  • Playbooks call VirusTotal, geo-locate source IPs, post Slack notifications, and update TheHive.
  • Added human-in-the-loop gates for containment actions (e.g., isolate host / reset password).
05
Validation & Metrics
  • Simulated Atomic Red Team techniques to validate rule + playbook coverage.
  • Measured mean-time-to-triage before & after — 80% reduction on repeatable playbook paths.
  • Documented runbooks and onboarding notes for future analysts.

Learnings

  • SOAR value compounds — the second playbook is 3× faster to build than the first because primitives are shared.
  • Case templates aligned to PICERL keep analyst investigations consistent without slowing them down.
  • Enrichment on ingest (not on demand) is what actually saves the time; analysts open cases already 70% resolved.
